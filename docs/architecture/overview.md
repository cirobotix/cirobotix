# Architektur – Überblick

cirobotix folgt einer pipeline-orientierten Architektur:

```text
Task / Work Order -> Context Assembly -> Prompt/Execution -> Output Checks -> Apply -> Format/Test
```

## Kernbausteine

- Blueprints
- Work Orders / Tasks
- Services
- Runner / Checker / Formatter
- Writers

## Nächste Schritte in der Doku

- Komponenten im Detail
- Datenfluss zwischen Modulen
- Architekturentscheidungen
